# 面试录 - 项目需求文档

## 项目概述

**项目名称**：面试录（Job Interview Tracker）

**项目目标**：开发一个简洁、实用的 iOS 求职记录管理应用，帮助求职者追踪和管理面试进度。

**技术栈**：
- UI 框架：SwiftUI
- 数据持久化：SwiftData (iOS 17+)
- 最低支持版本：iOS 17.0
- 开发语言：Swift 5.9+

---

## 核心红线（必须严格遵守）

### ✅ 必须遵守
1. **纯本地运行**：所有数据必须存储在用户手机本地
2. **禁止云端数据库**：严禁使用 Firebase、CloudKit 等
3. **禁止用户系统**：严禁添加登录/注册功能
4. **禁止网络请求**：严禁引入 Alamofire 等网络库
5. **原生 UI**：仅使用 SwiftUI，不使用 UIKit
6. **无第三方库**：不使用第三方 UI 库

### ❌ 严禁事项
- 添加任何形式的网络请求
- 使用云端数据库同步
- 引入第三方依赖
- 使用 UIKit 组件

## 功能需求

### 1. 数据结构

#### JobApplication（求职申请主表）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | ✅ | 唯一标识符 |
| companyName | String | ✅ | 公司名称 |
| positionName | String | ✅ | 岗位名称 |
| status | ApplicationStatus | ✅ | 整体状态 |
| appliedDate | Date | ✅ | 投递日期 |
| location | String | ❌ | 工作地点 |
| salaryRange | String | ❌ | 薪资范围 |
| notes | String | ❌ | 备注 |
| createdAt | Date | ✅ | 创建时间 |
| updatedAt | Date | ✅ | 最后更新时间 |

#### InterviewRound（面试轮次子表）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | ✅ | 唯一标识符 |
| roundName | String | ✅ | 轮次名称（支持自定义） |
| interviewDate | Date | ✅ | 面试日期 |
| interviewer | String | ❌ | 面试官姓名 |
| result | InterviewResult | ✅ | 本轮结果 |
| feedback | String | ❌ | 面试反馈 |
| createdAt | Date | ✅ | 创建时间 |

#### ApplicationStatus（求职状态枚举）
| 状态 | 说明 | 颜色 |
|------|------|------|
| applied | 已投递 | 蓝色 |
| writtenTest | 笔试 | 紫色 |
| interviewing | 面试中 | 橙色 |
| offer | Offer | 绿色 |
| rejected | 已拒绝 | 灰色 |
| failed | 未通过 | 红色 |

#### InterviewResult（面试结果枚举）
| 结果 | 说明 |
|------|------|
| pending | 待定 |
| passed | 通过 |
| failed | 未通过 |
| cancelled | 已取消 |

---

### 2. 页面功能

#### 首页（Dashboard）
- 使用 `List` 展示所有求职记录
- 每个条目显示：
  - 公司名（加粗）
  - 岗位名
  - 当前整体进度标签（颜色区分）
- 支持侧滑删除
- 支持点击进入详情页
- 右上角"+"按钮添加新记录

#### 详情页（Detail View）
- 顶部：公司和岗位信息
- 中间：面试轮次列表（时间轴样式）
- 底部：添加面试记录按钮
- 支持编辑求职信息
- 支持删除面试轮次

#### 添加求职记录页面
- 表单输入：
  - 公司名称（必填）
  - 岗位名称（必填）
  - 投递日期（默认今天）
  - 工作地点（可选）
  - 薪资范围（可选）
  - 备注（可选）
- 表单验证
- 保存按钮

#### 添加面试轮次页面
- 表单输入：
  - 轮次名称（必填，支持自定义）
  - 面试日期（必填）
  - 面试官（可选）
  - 本轮结果（必填）
  - 面试反馈（可选）
- 保存按钮

---

### 3. 交互细节

#### 状态颜色标识
| 状态 | 颜色 | 用途 |
|------|------|------|
| 进行中 | 黄色/橙色 | 已投递、笔试、面试中 |
| 成功 | 绿色 | Offer |
| 失败 | 红色 | 未通过 |
| 结束 | 灰色 | 已拒绝 |

#### 动画效果
- 页面切换动画
- 列表项删除动画
- 状态变更动画
- 按钮点击反馈
